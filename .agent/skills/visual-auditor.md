# Skill: Visual Auditor (Stitch Sync)
**Description**: Acts as a bridge between the Google Stitch visual deconstruction and the React/Tailwind code in Antigravity.

## Core Instructions
1. **Source of Truth**: Always reference the `DESIGN.md` file generated by the `/design-component` workflow before suggesting UI changes.
2. **Component Mapping**: When the user asks to modify a section (e.g., "Change the Hero"), identify the corresponding Section ID from the Stitch project.
3. **Visual Consistency**:
    - **Spacing**: Ensure all margins and padding match the "Atomic Breakdown" tokens.
    - **Typography**: Force the use of the font-family and weights identified during the last Stitch capture.
    - **Colors**: Never use arbitrary HEX codes; only use the palette extracted into the project's design system.

## Action Steps
- **Audit Mode**: If the user asks "Does this match the design?", use the `playwright` tool to take a screenshot and compare the local CSS values against the Stitch deconstruction.
- **Fix Mode**: If a drift is detected (e.g., a button is too rounded), automatically suggest a Tailwind refactor to bring it back in line with the Stitch "Source of Truth."

## Goal
To ensure that the "Code" in Antigravity and the "Visual Sections" in Stitch remain 100% synchronized at all times.
